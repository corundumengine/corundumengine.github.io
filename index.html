The Project

Built for the love of CRPGs

Corundum is a bespoke engine: one genre, one game at a time, built from scratch in C++23. Engine, tools, and game grow together. One rule decides everything: C++ defines what can happen; JSON defines what does happen.

Follow along as the engine, tools, and game take shape.

The Vision

What's Being Built

📦

Spritepacker

CLI tool that packs individual sprite images into texture atlas PNGs and companion JSON metadata — ready for the engine to consume.

Complete
🌎

Worldsmith

World map creation tool with native Azgaar's Fantasy Map Generator import. Paint terrain, place settlements, define states, and export tilemap chunks for the engine.

In Development
🗣️

Talesmith

Handles dialogue and quests. A node-graph editor for branching conversations supports Talk, Choice, and Event nodes with conditions, variable mutations, sequencing, and event hooks, plus quest editing for tracking objectives and story progression.

In Development
🖼️

Spritesmith

ImGui-based editor for creating and managing sprite sheets, animation frames, and metadata directly within the engine's toolset.

In Development
🧱

Tilesmith

Visual tilemap editor for constructing 2D isometric worlds tile by tile — collision shapes, portals, layers, all integrated with the engine's renderer.

In Development
⚙️

2D Rendering

Hardware-accelerated 2D sprite and tilemap rendering via Sokol, with Metal, D3D11, and GLCore backends — targeting macOS, Windows, and Linux via GLFW.

In Development
⚔️

CRPG Systems

Branching dialogue and staged quests run in the engine today. Combat, items, and character progression are next.

In Development
📜

Data-Driven Design

Dialogue, quests, tilemaps, sprite sheets, and configuration are JSON — authored by the tools, readable in a text editor, validated at load.

In Development
🏰

Project Keystone

A 2D CRPG built entirely on Corundum Engine — the game the whole forge exists to ship. Story, world, and systems taking shape alongside the engine.
