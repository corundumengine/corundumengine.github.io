I'm building a custom game engine in C++23 to make a role-playing game. The reasonable choice would have been to use an existing one.

Corundum Engine is what came out of that decision — a 2D engine built from scratch for CRPGs, with cross-platform rendering and RPG systems in progress.

## What's Been Done So Far

The first public release is the **Spritepacker CLI** — an open-source texture atlas tool that packs individual sprite images into one or more atlas PNGs. It's available now on [GitHub](https://github.com/corundumengine/spritepacker).

Active development is underway on:

- **Core rendering** — 2D sprite and isometric tilemap rendering
- **Developer tools** — ImGui-based sprite sheet editor, tilemap builder, and world importer
- **CRPG systems** — Rules, quests, dialogue trees, and character progression
- **Dialogue system** — A simple dialogue system in place

I hope to start on the quest system later this summer.

## What to Expect

Posts will cover the engine's progress, the decisions behind it, and eventually the game I'm building on top of it.
