Welcome to the official development blog for the **Corundum Engine** — a custom 2D game engine written in C++23, purpose-built for classic role-playing games.

## What Is Corundum Engine?

The Corundum Engine is an in-development 2D game engine designed from the ground up for building CRPGs. It uses [Sokol](https://github.com/floooh/sokol) for cross-platform hardware-accelerated rendering (Metal on macOS, D3D11 on Windows, GLCore on Linux) and [GLFW](https://github.com/glfw/glfw) for windowing. Every part of the engine — from the rendering pipeline to the RPG systems — is being built in-house.

## What's Been Done So Far

The first public release is the **Spritepacker CLI** — an open-source texture atlas tool that packs individual sprite images into optimized atlas PNGs. It's available now on [GitHub](https://github.com/corundumengine/spritepacker).

Active development is underway on:

- **Core rendering** — 2D sprite and tilemap rendering via Sokol
- **Developer tools** — ImGui-based sprite sheet editor, tilemap builder, and world importer
- **CRPG systems** — Rules, quests, dialogue trees, and character progression

## What to Expect

This blog will cover development progress, technical deep-dives into engine architecture, design decisions, and eventually, details about the CRPG we're building on top of the engine.

If you're interested in following along, check out the [roadmap](/#roadmap) and star the project on [GitHub](https://github.com/corundumengine).
